Updated to v! (Saturday, May 17, 2014)


With the genuine wish of providing a truly secure browsing experience to all ScriptSafe users, this update focuses on implementing something that has been largely missing in previous versions: true inline script blocking.

Compelete credits for this update goes to gorhill and the HTTP Switchboard extension.

Without further ado, here are the additions/changes to ScriptSafe in this release:
  • v1.0.6.17:
    • implemented true inline script blocking (credits to gorhill and the HTTP Switchboard project)
I recognize and love to a great extension when I see one, and I would recommend ScriptSafe users to check out HTTP Switchboard, as it features granular control over what what pages load.

Over the past few months, I have been working on revamping ScriptSafe, implementing language support, and with the help of several translators, have German, Spanish, Brazilian, Chinese, and English (US and British) translations integrated. I will be considering moving the project from Google Code to Github for increased transparency and collaboration potential. If you are feeling brave, you may download v1.0.7.0 alpha here. Please make a back-up of your settings.

Thank you, and enjoy.



Just as a note: ScriptSafe does not have any backdoors, nor does it contact any third-party server at any point in time. ScriptSafe was created by one ordinary guy (me), lovingly coded in my bedroom, without any intention other than to give you back control of the Internet.
